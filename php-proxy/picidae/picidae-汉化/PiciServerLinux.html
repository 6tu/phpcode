在Linux上安装pici-Server（测试版）¶

设置基础¶

您需要在计算机上安装以下程序才能成功运行pici服务器：

  • Apache Webserver
  • PHP（包括GD和mcrypt支持）
    • 确保在php中禁用安全模式。
  • MySQL

下载pici-server软件¶

下载pici-server软件。您可以下载其中一个每夜快照或获取subversion访问：

  • pici-server_snapshot.zip
  • pici-server_snapshot.tar.gz
  • svnAccess

将pici服务器软件解压缩到您的Web服务器目录中。请确保路径不包含皮皮或id科（见http://www.picidae.net/questions-and-answers/）。最好的办法是将pici服务器放在你的web服务器的根目录下。

为缓存文件夹设置权限¶

允许您的www用户在CACHE文件夹中创建文件。您可以通过shell更改权限： 

chmod -R 777 Path2YourPiciServer / CACHE

从源安装¶

安装外部需要的库和工具：

下载web2pici源代码：

  • web2pici_snapshot.tar.gz

编辑文件web2pici.cpp，并取消注释操作系统的第45行的包含路径。 

/ * ------------------------------------------------ ------------

//＃include <dom / html_misc.h> // < -  use this for Suse and Mandriva

。/配置

cp web2pici Path2YourPiciServerDirectory / ADMIN / web2pici

设置一个虚拟屏幕¶

在服务器上 安装虚拟Frambuffer程序Xvfb。

库名称 Debian 使用yast2 Fedora 请在此处添加您的分发
Xvfb apt-get install xvfb

在Shell中打开这种虚拟屏幕类型。（将验证文件的路径插入到pici服务器。） 

Xvfb：10 -auth Path2pici-server / ADMIN / xvfb.conf -screen 0 1200x5050x24＆

在时刻，你需要并行呈现的每张照片都是一个自己的屏幕。（这将是很好的不需要，但是，如果你正在运行一个屏幕只有任务被调度和处理一个接一个。）我遇到以下RAM消耗。一个屏幕是安装需要大约30MB RAM。每个打开的窗口需要大约20MB的RAM。

您可以根据需要设置任意数量的虚拟屏幕，每个虚拟屏幕连续编号。

让您的用户访问shell脚本¶

为了有权访问显示和窗口系统，您需要使用启动显示的用户的用户帐户。在配置文件中配置拥有显示会话的用户。因此，我们需要给运行php sudo权限的用户提供脚本。为此，我们不必在计算机上以纯文本形式存储密码，我们赋予它在sudo模式下无密码地执行python脚本的权利。

运行Web服务器和php的默认用户在每个发布中的命名方式不同：

  • Debian：www-data
  • Ubuntu：www-data
  • Suse：wwwrun

找出你的php用户如何调用>>

在服务器上运行以下php脚本，可以看到当前 PHP 用户的名称 

<?php

$uname = shell_exec ('whoami');
echo ("用户名是: <b>$uname </b>");

?>

重要的是要知道你的PHP用户的名字，因为你必须给它sudo权限在visudo文件中生成图片。

打开Shell（也称为终端）并键入： 

sudo visudo

（这将打开你的标准shell编辑器中的sudoers文件：“vi”在我的case）

为了使用“visudo”命令，你需要包“sudo”。

将以下行添加到visudo文件。注意路径指向你的shellscript mkpicture.sh！（在此示例中给出的路径只是正确的，如果您在Web服务器的主目录中安装了pici-server）。在示例中使用www-user的名称，而不是“www” 

www ALL =（ALL）NOPASSWD：Path2YourPiciServer / ADMIN / mkpicture_xvfb.sh

编辑器vi不是那么容易处理（这里简短的解释）：

  1. 用光标移动到要插入新行的位置。
  2. 按“i”
    • 使用箭头键将光标移动到最后一行的末尾，然后按回车插入新行。
    • 然后插入新行。 
      www ALL =（ALL）NOPASSWD：Path2YourPiciServer / ADMIN / mkpicture_xvfb.sh
  3. 按“退出”键退出插入模式。
  4. 键入以下内容以保存文件 
    ：wq
  5. 点击进入和你的完成。

创建MySQL数据库¶

确保MySQL已启动并正在运行。然后安装数据库。有很多方法来安装MySQL数据库。以下是通过终端的快速描述： 

mysql -u root -p

然后插入您在安装中使用的MySQL密码。在MySQL控制台类型中： 

创建数据库picidae;
出口;

设置配置文件¶

将配置文件“ADMIN / sample.config.php”复制到“ADMIN / config.php”，并在配置文件中设置正确的路径。

创建数据库表¶

要设置pici-server的数据库结构，您应该在Web浏览器中调用文件ADMIN / setup.php并按照安装说明进行操作。
（如果pici-server位于Web服务器的主目录中，则为http：//localhost/ADMIN/setup.php）

安装脚本还将检查您是否正确配置了pici-server，并会在出现错误时发出警告。如果收到错误消息，请解决问题并重新调用设置脚本，直到收到最终成功消息。

pici-server成功安装¶

现在你可以尝试访问你的pici服务器，它应该是启动和运行。
