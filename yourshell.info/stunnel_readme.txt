动态网加密代理使用说明

1　下载sproxy.zip。
2　解压缩到同一个目录下。共有5个文件：libeay32.dll, libssl32.dll, stunnel-4.00.exe, stunnel.conf, readme.txt。

自动设置：
3　下载自由门软件到同一个目录下。
4　启动自由门，找到服务器后，在“代理”页面下，点击“设置密道”键，IE的代理服务器会被设置成127.0.0.1:8081。在IE地址栏中可输入海外任意网站直接浏览。点击“取消密道”键，可取消这个代理设置。

手动设置：
3　用自由门软件、在线查询或其他方法找到一个动态网结点后，确认可以用https访问这个结点。
4　用写字板打开stunnel.conf，把“动态网结点IP”换成您刚找到的这个有效IP或域名。如果找到的IP是67.33.117.94，那么改过以后，这一行由“connect = 动态网结点IP:443”变成“connect = 67.33.117.94:443”。另外注意粘贴的时候不要多加了一个空行。
5　双击stunnel-4.00.exe开始运行。
6　把IE的代理服务器设置成127.0.0.1:8081。注意在 IE 设置 Http 代理时，“ IE |工具| Internet 选项|连接”有两种：“局域网（LAN）设置”和“拨号和虚拟专用网络设置”。“局域网（LAN）设置”对局域网生效；“拨号和虚拟专用网络设置”对拨号网络生效。但有时“拨号和虚拟专用网络设置”设置的代理会失效，不通过代理直接访问网站。原因可能是多种上网环境，显式的是拨号上网，还有隐式的 NAT、DHCP（有些是这样）。通过隐式的方式出去，就出现“拨号和虚拟专用网络设置”设置的代理失效。可以将“局域网（LAN）设置”和“拨号和虚拟专用网络设置”都设置代理来避免这种情况。

然后您访问任何网站都是通过动态网结点，在您的计算机和动态网结点之间的传输全部是加密的。

技术说明：

Stunnel 是一个源代码开放的网络传输软件。她在应用层和TCP层之间，利用SSL技术，对所有网络数据流的应用层加密。理论目可以加密任何TCP协议。客户与服务器间的防火墙只能看到双方的IP和端口。这里把动态网的一次代理做了加密，速度会比CGI代理更快。

中国的防火墙只能看到加密后的数据，不知道用户传输的内容。
